\section{Exercise 1: Critical Paper Review}
%==============================================================================

\subsection{Q1: Primary Contributions}

The paper addresses an under-explored problem in NLP: detecting Patronising and Condescending Language (PCL) towards vulnerable communities in media discourse.
The authors introduced the \textit{Don't Patronize Me!} dataset, containing 10637 paragraphs concerning vulnerable social groups, annotated for both PCL presence and specific PCL categories.
The authors evaluate several baseline models, including SVM-based methods, BiLSTM, and BERT variants, with RoBERTa-base achieving the best performance (F1 = 70.63\%) on binary PCL classification.
The authors also proposed a taxonomy of PCL categories, aimed to help better understand the different forms of PCL and to guide future research.

\subsection{Q2: Technical Strengths}

\begin{itemize}[noitemsep]
    \item The dataset draws from news sources across 20 English-speaking countries and covers 10 distinct vulnerability keywords, ensuring diversity and representativeness.
    \item The two step annotation scheme, which first identifies PCL presence and then categorises PCL spans, enables both binary classification and fine-grained category analysis.
    \item The annotation process used a rigorous two-step methodology, where the first step involved identifying the presence of PCL and the second step involved categorising the type of PCL, which allows for more detailed analysis.
    \item The paper establishes clear benchmarks through comprehensive evaluation of both traditional methods (SVM, BiLSTM) and transformer-based models (BERT, RoBERTa, DistilBERT).
\end{itemize}

\subsection{Q3: Key Weaknesses}

\begin{itemize}[noitemsep]
    \item The dataset is relatively small for training large transformer-based models, particularly for underrepresented categories (e.g., \textit{"The poorer, the merrier"} contains only 64 samples), risking overfitting and limiting generalisation.
    \item The experiments use only paragraph-level classification, the available span-level annotations are not utilised for training, potentially limiting the models' ability to learn fine-grained PCL patterns.
\end{itemize}

\subsubsection{Analysis}

The basic statistical profiling reveals concrete structural challenges in the PCL dataset:

\begin{enumerate}[noitemsep]
    \item \textbf{Severe Class Imbalance.} Exactly 7{,}581 of the 8{,}375 training samples (90.5\%) are Non-PCL, leaving only 794 samples (9.5\%) as positive PCL examples, yielding a $\sim$9.5:1 imbalance. A trivial classifier predicting ``Non-PCL'' for every input would achieve 90.5\% accuracy while having zero utility.

    \item \textbf{Within-PCL Label Sparsity.} Among the 7 PCL categories (Labels 0--6), Label\_0 is the most frequent (574 samples, 6.85\%) and Label\_5 the second most (363, 4.33\%), while Label\_6 is extremely rare with only 29 instances (0.35\%). For the binary task all labelled samples are treated uniformly as PCL\,=\,1, but this internal skew is relevant if fine-grained multi-label classification is attempted later.

    \item \textbf{Label Co-occurrence.} Of the 794 PCL samples, 551 (69.4\%) carry more than one label, with individual samples carrying up to 5 labels simultaneously. When a text is patronising, it typically exhibits multiple PCL patterns at once, indicating PCL is a multi-faceted linguistic phenomenon rather than a single discrete signal.

    \item \textbf{Right-Skewed Token Distribution.} The distribution has a median of 38 tokens, a mean of 44.18 (std 26.85), and a maximum of 820 tokens. Most samples are short-to-medium paragraphs, but a long tail of substantially longer passages must be accommodated.
\end{enumerate}

\subsubsection{Impact Statement}

\begin{itemize}[noitemsep]
    \item \textbf{Metric Selection.} With 90.5\% Non-PCL, accuracy is actively misleading. Optimisation must target \textbf{Macro-F1} with explicit tracking of the positive-class (PCL) F1. A threshold sweep on the dev set is also necessary, as the default 0.5 decision boundary is biased toward the majority class.

    \item \textbf{Loss Function.} Standard Binary Cross-Entropy will be overwhelmingly biased toward predicting ``Non-PCL''. \textbf{Weighted Binary Cross-Entropy} with \texttt{pos\_weight}\,$\approx$\,9.5 (reflecting the 9.5:1 ratio) or \textbf{Focal Loss} must be used to redirect gradient updates toward the rare PCL cases.

    \item \textbf{Sampling Strategy.} With a 1:9.5 ratio, a random mini-batch of 32 samples contains on average only $\sim$3 PCL examples, resulting in an extremely noisy training signal. \textbf{Stratified sampling} per batch is required to guarantee consistent PCL representation across training steps.

    \item \textbf{Sequence Length.} The median of 38 tokens sits well within 128 subword tokens for most transformer tokenisers, making \texttt{max\_length=128} sufficient for the bulk of the data. However, because the maximum reaches 820 tokens, \texttt{max\_length=256} is the safer default to avoid truncating the longer, elaborately structured PCL paragraphs.

    \item \textbf{Embedding Strategy.} The 26{,}897 unique tokens justify using a \textbf{pre-trained embedding} (e.g.\ GloVe or a BERT sub-word tokeniser) rather than training word embeddings from scratch on this small corpus.
\end{itemize}

\subsubsection{Analysis}

\begin{enumerate}[noitemsep]
    \item \textbf{N-gram Patterns — Vulnerability-Framing Phrases.} Top bigrams in both classes are dominated by generic English function-word pairs (``of the'', ``in the''), which carry no discriminative power on their own. The signal emerges at the trigram level. Normalising by class size, PCL text shows a \textbf{4.2$\times$ enrichment} for ``those in need'' (0.050 per PCL sample vs 0.012 for Non-PCL), and ``people in need'' appears in PCL's top 10 but not in Non-PCL's. These phrases construct a narrative of passivity and dependency, which is a hallmark of patronising language. Non-PCL trigrams, by contrast, are factual and geopolitical: ``the united states'', ``the number of'', ``as well as''.

    \item \textbf{Stop Word Density — PCL Uses More Elaborate Syntax.} PCL texts have a mean stop-word density of \textbf{48.47\%} (median 48.8\%) versus \textbf{45.01\%} (median 45.2\%) for Non-PCL, representing a gap of \textbf{3.46 percentage points} (std $\approx$9.7\% in both classes, confirming the difference is systematic). The higher density reflects the clause-heavy sentence structures of patronising language (e.g.\ \textit{``they are the ones who are in need of our help''}) compared to the more direct, noun-heavy style of factual reporting.

    \item \textbf{Distinctive Vocabulary (Per-Sample Rates).} After normalising for class size, several words are significantly over-represented in PCL text: ``need'' (2.2$\times$), ``poor'' (2.2$\times$), ``children'', ``homeless'', and ``help'' appear in the PCL top-10 but are absent from the Non-PCL top-10 entirely. This confirms a PCL lexical fingerprint centred on \textit{vulnerability, dependency, and aid}. Non-PCL top words (``immigrants'', ``vulnerable'', ``disabled'') cover the same topic domain but without the emotional framing.
\end{enumerate}

\subsubsection{Impact Statement}

\begin{itemize}[noitemsep]
    \item \textbf{Retain Stop Words for Bag-of-Words Models.} The 3.46\% higher stop-word density in PCL text means connective and auxiliary words actively participate in PCL phrasing. Trigrams like ``in need of'', ``to those in'', and ``those in need'' depend entirely on stop words. For any TF-IDF baseline, stop words must be \textbf{retained} with \texttt{ngram\_range=(1,\,3)}.

    \item \textbf{N-gram Features for Classical Models.} The distinctive trigrams are strong discriminative signals that a unigram model would miss entirely. Any SVM or logistic regression baseline should be trained on \textbf{TF-IDF weighted 1--3-grams}.

    \item \textbf{Class-Size Normalisation in Analysis.} Raw frequency counts are dominated by the 9.5$\times$ larger Non-PCL class. Per-sample enrichment ratios must be computed for any fair lexical comparison, as demonstrated above.

    \item \textbf{Pre-trained Embeddings Domain Alignment.} The PCL vocabulary is centred on humanitarian/social-narrative themes. Embeddings pre-trained on news or social-media corpora will better capture these semantic relationships than generic web-crawl embeddings.

    \item \textbf{Keyword-Based Baseline.} The identified enriched vocabulary enables a simple TF-IDF logistic regression baseline, providing a useful lower bound to verify that the trained neural model learns beyond surface lexical patterns.
\end{itemize}

\section{Exercise 3: Proposed Approach}
%==============================================================================

\subsection{Proposed Approach}

We propose to fine-tune \textbf{RoBERTa-base} (Liu et al., 2019) as the core encoder, augmented with three targeted modifications designed around the structural challenges identified in the EDA: a dual-head multi-task architecture that exploits the dataset's fine-grained label taxonomy, a compound strategy to mitigate the 9.5:1 class imbalance, and context-aware input formatting that injects the vulnerability keyword as an explicit signal.

\subsubsection{Model Architecture}

We plan to attach two lightweight linear heads to the \texttt{[CLS]} token representation of a fine-tuned RoBERTa-base encoder. The \textbf{primary binary head} will produce the PCL / Non-PCL decision used at inference time. In addition, an \textbf{auxiliary multi-label head} will simultaneously predict the 7 fine-grained PCL category labels defined by P\'{e}rez-Almendros et al.\ (2020) during training. The two heads share the same encoder and are trained jointly under the combined loss:
\begin{equation}
    \mathcal{L} = \mathcal{L}_{\text{focal}} + \lambda \cdot \mathcal{L}_{\text{aux}}
\end{equation}
where $\mathcal{L}_{\text{focal}}$ is Binary Focal Loss (Lin et al., 2017) on the binary head and $\mathcal{L}_{\text{aux}}$ is Binary Cross-Entropy on the auxiliary head. The auxiliary head will be discarded at inference; it serves purely as an additional supervision signal during training.

\subsubsection{Data Preprocessing}

Each input will be formatted as \texttt{[keyword] paragraph\_text}, prepending the vulnerability keyword (e.g.\ \textit{homeless}, \textit{immigrants}) as a prefix token. This injects the topic context directly into the sequence without any architectural overhead, aligning the model's input with the annotation process, in which annotators were presented with paragraphs together with their associated keyword. We will use \texttt{max\_length=256} tokens to accommodate the long-tail of longer paragraphs identified in the EDA while keeping memory usage manageable.

\subsubsection{Class-Imbalance Mitigation}

We propose three complementary strategies to address the 9.5:1 class imbalance:

\begin{enumerate}[noitemsep]
    \item \textbf{Weighted batch sampling} to oversample PCL positives in every mini-batch, ensuring a consistent minority-class gradient signal throughout training.
    \item \textbf{Focal Loss} (Lin et al., 2017) to down-weight easy-to-classify negatives at the loss level, directing gradient updates toward the harder PCL examples.
    \item \textbf{Post-hoc decision threshold optimisation} by sweeping thresholds on the official dev set after training, replacing the default 0.5 boundary which is biased toward the majority class.
\end{enumerate}

\subsubsection{Optimisation Strategy}

We will use AdamW with a \textbf{two-group learning rate schedule}: a lower rate for the pre-trained encoder layers to avoid catastrophic forgetting, and a higher rate for the newly initialised classification heads. A cosine learning rate schedule with linear warmup will be applied, alongside gradient accumulation to achieve a larger effective batch size without requiring additional GPU memory. Early stopping on dev F1 will be used to prevent overfitting.

\subsection{Rationale and Expected Outcome}

\subsubsection{Rationale}

\textbf{Multi-task learning via the auxiliary head.}
The \textit{Don't Patronize Me!} dataset provides 7 fine-grained PCL category labels, including \textit{Unneeded Rescue}, \textit{Shallow Solution}, and \textit{Compassion}, which encode structurally distinct manifestations of patronising language. We hypothesise that training the model to simultaneously predict these categories will force the shared encoder to learn richer, semantically grounded representations of PCL, rather than relying on shallow surface correlations. This is particularly important given the small size of the positive class (794 examples), where additional supervision signal is valuable.

\textbf{Focal Loss and weighted sampling.}
The EDA showed that standard Binary Cross-Entropy on this dataset would concentrate gradient updates overwhelmingly on the 90.5\% Non-PCL majority, pushing the model toward the trivial all-negative prediction. Focal Loss addresses this at the loss level, while the weighted sampler ensures each batch reflects the minority class at the data level. Using both in tandem provides two independent, complementary mechanisms against the imbalance.

\textbf{Keyword prefix injection.}
The lexical EDA demonstrated that discriminative PCL phrases (e.g.\ ``those in need'') co-occur with specific vulnerability topics. Prepending the keyword makes this context explicit and enables the model to learn topic-conditioned PCL patterns. This is a zero-parameter improvement that also aligns the input with how the dataset was annotated.

\textbf{Threshold optimisation.}
The default 0.5 decision threshold is calibrated for balanced classes. For an imbalanced task like PCL detection, sweeping the threshold on the dev set to maximise F1 is expected to yield a meaningful free improvement in the precision--recall trade-off.

\subsubsection{Expected Outcome}

The shared task baseline RoBERTa-base model achieves $\text{F1} = 0.48$ on the official dev set (P\'{e}rez-Almendros et al., 2020). We expect our proposed modifications to deliver a substantial improvement, primarily by increasing the model's recall on PCL-positive examples through better imbalance handling, while the auxiliary multi-task objective and keyword context injection should additionally improve precision by anchoring predictions in structured PCL patterns.

\section{Exercise 4: Model Training}
%==============================================================================

The proposed approach was implemented in PyTorch using the HuggingFace \texttt{transformers} library. The full training code and model weights are available in the \texttt{BestModel/} folder of the repository.

\noindent\textbf{Repository Link:} \url{https://github.com/LemonPhase/pcl-detection}

\subsection{Implementation Details}

\subsubsection{Architecture}

RoBERTa-base was loaded from the \texttt{FacebookAI/roberta-base} checkpoint. Two linear heads were attached to the \texttt{[CLS]} token after a shared Dropout layer ($p=0.2$): a binary head \texttt{Linear(768, 1)} and an auxiliary 7-label head \texttt{Linear(768, 7)}. Both heads were initialised with small normal weights (std $= 0.02$) and zero biases for training stability.

\subsubsection{Data Split and Evaluation Protocol}

The official training set (8,375 samples) was used for training and the official dev set (399 samples) was used as the held-out evaluation set throughout all experiments; no internal dev split was created, as the official dev set provides a stable benchmark comparable across runs. The official test set (3,832 samples, labels withheld) was used only for final prediction.

After training, a threshold sweep over $t \in [0.05, 0.95]$ (step 0.01) was run on the dev set to select the decision boundary maximising PCL-class F1.
